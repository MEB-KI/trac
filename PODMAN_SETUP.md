To set up the development containers without using a pod, use a private Podman network instead. This keeps the database isolated from the host while allowing the backend and web containers to communicate using container DNS names.

Here's the sequence of bash/Podman commands. Run them in order from the project root (`trac`):

```bash
# Create named volumes for data persistence
podman volume create trac-postgres-data

# Create a private network for the service containers
podman network create trac-dev-net


# Run the PostgreSQL database container on the network
cd database

## TODO: the health-cmd hardcodes user and database, fix that
podman run -d --name db --network trac-net \
  --network-alias db \
  --health-cmd "pg_isready -U timeusediary_user -d timeusediary_db" \
  --health-interval 5s \
  --health-timeout 5s \
  --health-retries 10 \
  --env-file ../backend.env
  -e POSTGRES_PASSWORD=systemuserpassword \
  -v ./initscript:/docker-entrypoint-initdb.d \
  -v trac-postgres-data:/var/lib/postgresql \
  postgres:18-alpine

cd ..

# Build the backend image (from the backend directory)
cd backend
  podman build -t trac-backend -f ./Dockerfile .
cd ..

podman volume create trac-backend-venv
podman volume create trac-uv-cache

# Run the backend container on the same network
podman run -d --name backend --network trac-net \
  -w /workspace/backend \
  -v .:/workspace/backend:Z \
  -v trac-backend-venv:/workspace/backend/.venv \
  -v trac-uv-cache:/root/.cache/uv \
  -p 8001:8000 \
  trac-backend

cd ..

# Run the web (Nginx) container on the same network
podman run -d --name web --network trac-net \
  -v ./frontend/src:/usr/share/nginx/html/report:ro \
  -v ./frontend/src/settings/tud_settings.js:/usr/share/nginx/html/report/settings/tud_settings.js:ro \
  -v ./dev_tools/docker/webserver_config/dev.docker.nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  -p 3001:3000 \
  nginx:1.30-alpine

# (Optional) Check container status
podman ps

# (Optional) View logs for a specific container
podman logs -f backend

# To stop and clean up
podman stop backend web db
podman rm backend web db
podman network rm trac-dev-net
podman volume rm trac-dev-postgres-data trac-dev-backend-venv trac-dev-uv-cache  # if you want to remove volumes
```

**Notes:**
- The database is not published to the host, so it stays isolated from the host network.
- The backend connects to the database using the container name `db`, matching `TUD_DATABASE_HOST=db` in `dev_tools/docker/backend_settings/.env.dev-docker`.
- The frontend is published on host port `3000`, and the backend on host port `8000`.
- If you later want to run e2e tests in the same network, add `--network trac-dev-net` to the e2e container and use `http://web:3000/report/` inside that container.
- Volumes persist data across runs; remove them if you want a fresh start.
