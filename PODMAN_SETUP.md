To set up a Podman pod and run the containers manually (without `podman-compose`), you'll need to handle building images, creating volumes, and running containers individually. This replicates the docker-compose.dev.yml setup for the main services (db, backend, web). The e2e service is skipped as it's for testing.

Here's the sequence of bash/Podman commands. Run them in order from the project root (trac):

```bash
# Create named volumes for data persistence
podman volume create trac-dev-postgres-data
podman volume create trac-dev-backend-venv
podman volume create trac-dev-uv-cache

# Build the backend image (from the backend directory)
cd backend
podman build -t trac-backend -f ../dev_tools/docker/backend.Dockerfile .
cd ..

# Create a pod to group the containers (exposing ports 3000 and 8000)
podman pod create --name trac-dev-pod -p 3000:3000 -p 8000:8000

# Run the PostgreSQL database container in the pod
podman run -d --pod trac-dev-pod --name db \
  --health-cmd "pg_isready -U timeusediary_user -d timeusediary_db" \
  --health-interval 5s \
  --health-timeout 5s \
  --health-retries 10 \
  -e POSTGRES_DB=timeusediary_db \
  -e POSTGRES_USER=timeusediary_user \
  -e POSTGRES_PASSWORD=timeusediary_password \
  -v trac-dev-postgres-data:/var/lib/postgresql/data \
  postgres:18-alpine

# Run the backend container in the pod (wait a moment for db to be healthy if needed)
podman run -d --pod trac-dev-pod --name backend \
  -w /workspace/backend \
  --entrypoint sh \
  -c "uv sync --dev && uv run gunicorn --reload -c /workspace/dev_tools/docker/gunicorn_conf.docker.py o_timeusediary_backend.api:app" \
  -v ./backend:/workspace/backend \
  -v ./dev_tools/docker:/workspace/dev_tools/docker:ro \
  -v trac-dev-backend-venv:/workspace/backend/.venv \
  -v trac-dev-uv-cache:/root/.cache/uv \
  -v ./dev_tools/docker/backend_settings/.env.dev-docker:/workspace/backend/.env:ro \
  trac-backend

# Run the web (Nginx) container in the pod
podman run -d --pod trac-dev-pod --name web \
  -v ./frontend/src:/usr/share/nginx/html/report:ro \
  -v ./dev_tools/docker/frontend_settings/tud_settings.dev-docker.js:/usr/share/nginx/html/report/settings/tud_settings.js:ro \
  -v ./dev_tools/docker/webserver_config/dev.docker.nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  nginx:1.30-alpine

# (Optional) Check pod and container status
podman pod ps
podman ps

# (Optional) View logs for a specific container
podman logs -f trac-dev-pod-backend  # or use container name

# To stop and clean up
podman pod stop trac-dev-pod
podman pod rm trac-dev-pod
podman volume rm trac-dev-postgres-data trac-dev-backend-venv trac-dev-uv-cache  # if you want to remove volumes
```

**Notes:**
- The backend build may take time on first run.
- The db healthcheck ensures PostgreSQL is ready before the backend starts (in compose, this is handled by `depends_on`).
- Volumes persist data across runs; remove them if you want a fresh start.
- Access the app at `http://localhost:3000/report/` (frontend) and `http://localhost:8000` (backend API).
- If you encounter issues (e.g., with paths or builds), ensure you're in the correct directory and that the referenced files exist.
